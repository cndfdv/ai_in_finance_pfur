# Детекция мошеннических транзакций

В этой лабораторной работе необходимо принять участие в соревновании курса на Kaggle и построить модель для выявления мошеннических карточных транзакций.

Подробное описание задачи, структура данных и правила отправки решений приведены на странице соревнования:

https://www.kaggle.com/competitions/detect-fraud-in-card-transactions-pfur

## Задача

Необходимо предсказать вероятность того, что карточная транзакция является мошеннической.

Тип задачи: **бинарная классификация**.

Целевая переменная: `is_fraud`.

## Метрика

Качество решения оценивается по метрике **ROC-AUC**.

## Формат решения

Для каждой транзакции из тестовой выборки необходимо предсказать вероятность мошенничества в диапазоне от 0 до 1.

Решение загружается на Kaggle в формате `submission.csv`.

## Оценивание

| Оценка | Значение ROC-AUC |
|---:|---:|
| 5 | ≥ 0.985 |
| 4 | 0.950–0.985 |
| 3 | 0.900–0.950 |
| 2 | 0.800–0.900 |
| 1 | 0.600–0.800 |
| 0 | < 0.600 или отсутствие сабмишена |
